# Фазы реализации: Onsite Backend

Разбивка задач [PDR](pdr.md) на 6 фаз; каждая фаза выполнялась по отдельному плану (планирование → реализация → верификация).


| Фаза | Название            | Задачи PDR                   | Содержание                                                                                                                                   | Веха     |
| ---- | ------------------- | ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- | -------- |
| 1    | Каркас (M0)         | T-01, T-02, T-03, T-18       | Fastify 5 + TS strict + Drizzle, Docker Compose (api/pg/minio), CI, semantic-release; схема БД, миграции, сид; `/health`, OpenAPI; логи pino | M0       |
| 2    | Auth и пользователи | T-04, T-05, T-06             | Логин, ротация refresh с отзывом семьи, logout; роли и авторизация; управление аккаунтами, сброс пароля                                      | —        |
| 3    | Заявки              | T-07, T-08, T-09             | CRUD заявок, назначение с историей `order_assignments`; конечный автомат статусов; append-only `order_events`                                | —        |
| 4    | Фото                | T-12, T-13                   | Staged-загрузка (multipart, `Idempotency-Key`, S3), зачистка сирот; выдача через 302 на presigned URL                                        | —        |
| 5    | Синк-протокол       | T-10, T-11, T-14             | Pull по курсору с tombstone и safety-lag; батч мутаций (идемпотентность, вердикты, конфликты); e2e «офлайн-смена»                            | **MVP**  |
| 6    | Продакшн (v1.0)     | T-15, T-16, T-20, T-17, T-19 | Регистрация устройств; push-worker (outbox, Expo, receipt'ы); метрики `/metrics` и алёрты; rate limiting; OWASP-чек и деплой self-host       | **v1.0** |


## Порядок и зависимости

- Фазы выполняются строго последовательно: 1 → 2 → 3 → 4 → 5 → 6.
- Фаза 4 (фото) идёт перед синком: T-11 (батч мутаций) зависит от T-12 — мутация `photo_add` коммитит staged-фото.
- Отличия от roadmap PDR (§8): этапы 2–3 разбиты по границе auth/заявки; стабилизация (этап 4 PDR) влита в фазу 5; фото вынесены перед синком по зависимостям.
